# INPUT FILES
#
# If the -simulation option is specified when calling surfsuite, the parameters listed
# under this particular simulation will be loaded. Otherwise the parameters under the
# first-listed simulation tag and "default" tag (in this order) will be adopted.
#
# The GADGET-2 snapshots must be sorted in one or several files of name
# [path_gadget]/[snapshot] OR
# [path_gadget]/[snapshot].0, [path_gadget]/[snapshot].1, ...
#
# The VELOCIraptor HDF5 groups must be stored in one or several files of name
# [path_velociraptor]/[snapshot][ext_groups] OR
# [path_velociraptor]/[snapshot][ext_groups].0, ...
#
# The VELOCIraptor HDF5 particles must be stored in one or several files of name
# [path_velociraptor]/[snapshot][ext_particles] OR
# [path_velociraptor]/[snapshot][ext_particles].0, ...
#
# OUTPUT FILES (generated by SurfSuite)
#
# The sorted particles (id, species, 3D-position, 3D-velocity) in binary format are in one
# or several files of name
# [path_surfsuite]/[snapshot][ext_sorted] OR
# [path_surfsuite]/[snapshot][ext_sorted].0, ... (same number as GADGET files)
# Whether one or several files are used depends on the number of particles. The maximum
# number of particles is specified in the global variable nparticles_per_sorted_file
# and is normally set to 10^8, such that the file sizes are limited to about 3.6GB.
# This sorting is just by particle id and has nothing to do with VELOCIraptor.
#
# The halo particles (id, position, velocity) in binary format are in one or
# several files of name
# [path_surfsuite]/[snapshot][ext_halos] OR
# [path_surfsuite]/[snapshot][ext_halos].0, ... (same number as VELOCIraptor files)
#
# The pointer list of the halos is stored in a single file
# [path_surfsuite]/[snapshot][ext_halolist]
#
# The additional path [path_analysis] is generated automatically by surfsuite, but left
# empty. It is meant to serve as a post-processing path for other code that can also
# access this parameter file, such as the private tool "surfanalysis".
